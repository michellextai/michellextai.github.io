Hunt Statement

To better support individuals in existing cancer support communities, we want to design a mobile-friendly environment for CSN. To give us design direction, we will first understand the user needs and patterns through user research, while noting the differences in user behavior and flow between mobile and web usage. We will conduct competitive analysis as a way to learn about usability of ACS competitors and incorporate those findings to optimize our mobile navigational design.

Client Need

CSN is a dynamic online community created by and created for people affected by cancer. There are approximately currently 65,000 active community users out of 170,000 registered users. They use the Cancer Survivors Network in order to post discussions and chat with other cancer patients. The client seeks to redesign the current website interface in order to better connect users to others with similar interests and experiences. As of now, users may find the site difficult to navigate and therefore cannot access features and discussions they may be looking for. This also lowers user retention and return rates, which the client seeks to increase with the redesign. Currently, 45% of the users visit the CSN website through mobile devices. With the growing number of tech-savvy users, a mobile-friendly interface is necessary to cater toward a wider range of media types. As the average cancer surviving patient is 65 years old, we also need to keep accessibility for elderly users in mind. ACS is planning to deploy a new website in 2018 and want to consider new design ideas before implementation. Therefore, the client has expressed little priority on pushing code changes to the website.
